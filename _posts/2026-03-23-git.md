title: How Does Git Work
---

The magic part of Git is that it can store so many commits at the same time but only few disk space is used. (e.g. If you use Git to manage a project about 1MB size, it may only use 1MB and a little space to store the projects and its so many commits.)
The two genius method Git use is reuse and delta.

### Reuse
Git generate all the file name using the content of this file. If the contents of two file is exactly the same, so are their file name. So when we create one new commit in out project, instead of creating new file to take more space, Git will compare the file names based SHA-1. If the file names is same, then Git will reuse the old file.

### Delta
When we change some files in one commit, Git can't reuse the old file. Cleverly, Git will only store the diff between those files. When we use specific file, Git will recreate the file from the original one and those deltas. What needs to be emphasized is the layer of delta is not unlimited. When the layer reach 50, Git will store the new file instead of storing the diff.
